# B2B план (кратко)

## Что изменено

- Убрана runtime-зависимость от `ingredients`: cart/order/search/product-form работают на единой товарной модели.
- Добавлены универсальные связи товар-товар (`related products`) для блока «С этим товаром берут».
- Админка поддерживает настройку связей: тип (`RECOMMENDED`/`ACCESSORY`/`COMPATIBLE`), порядок (`sortOrder`), заметку совместимости.
- `createOrder` и snapshot заказа переведены на товарные позиции; добавлены regression-тесты.
- Добавлен legacy fallback чтения старых заказов в admin-panel (старый и новый формат snapshot).
- Обновлены seed/документация под новую модель.

## Как теперь работает цена

- Источник цены: товар/вариант товара.
- Отображение цены в витрине, карточке, корзине и checkout управляется `priceMode` из админки.
- Если `priceMode = request`, цена скрыта в UI и используется сценарий «запросить/отправить заказ менеджеру».

## Как теперь работает заказ

- Корзина и checkout работают только через товарные позиции.
- Сопутствующие товары добавляются как обычные позиции корзины.
- Итоги корзины и данные заказа считаются по товарной модели.

## Как работает распределение по менеджерам

Приоритет маршрутизации заявки:

1. Точное назначение по `productId`.
2. Назначение по `categoryId`.
3. Fallback менеджер tenant (`handlesAllCategories=true` или `isPrimary`).

Требование: у каждого tenant должен быть хотя бы один fallback-менеджер.
